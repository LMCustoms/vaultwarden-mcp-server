# Watchtower Auto-Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Server-wide auto-deployment via Watchtower across LMCustoms infrastructure, with GHCR image pipelines, webhook triggers, email notifications, and project-registry integration.

**Architecture:** Watchtower runs on each server, polling Docker registries for image updates. Custom LMCustoms services publish images to GHCR on GitHub Release. GitHub Actions triggers Watchtower immediately via SSH+curl for our own services. Watchtower notifies via email and POST to project-registry for deployment tracking.

**Tech Stack:** Docker, Watchtower (containrrr/watchtower), GitHub Actions, GHCR, Go (project-registry), PostgreSQL, Stalwart SMTP

**Spec:** `docs/superpowers/specs/2026-04-12-watchtower-auto-deployment-design.md`

## Task 5: Create Shared notify-watchtower Workflow

**Repo:** `LMCustoms/.github`
**Files:**
- Create: `.github/workflows/notify-watchtower.yml`

- [ ] **Step 1: Create the shared workflow**

Create `.github/workflows/notify-watchtower.yml`:

```yaml
name: "Notify Watchtower"

on:
  workflow_call:
    secrets:
      DEPLOY_HOST:
        required: true
      DEPLOY_USER:
        required: true
      DEPLOY_SSH_KEY:
        required: true
      WATCHTOWER_HTTP_API_TOKEN:
        required: true

jobs:
  notify:
    name: Trigger Watchtower Update
    runs-on: ubuntu-latest
    if: github.event_name == 'release' || github.event_name == 'workflow_dispatch'

    steps:
      - name: Trigger Watchtower via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.DEPLOY_HOST }}
          username: ${{ secrets.DEPLOY_USER }}
          key: ${{ secrets.DEPLOY_SSH_KEY }}
          script: |
            echo "Triggering Watchtower update check for ${{ github.repository }}..."
            HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" \
              -X POST "http://127.0.0.1:8080/v1/update" \
              -H "Authorization: Bearer ${{ secrets.WATCHTOWER_HTTP_API_TOKEN }}")
            if [ "$HTTP_CODE" -ge 200 ] && [ "$HTTP_CODE" -lt 300 ]; then
              echo "Watchtower update triggered (HTTP $HTTP_CODE)"
            else
              echo "Warning: Watchtower trigger failed (HTTP $HTTP_CODE) — non-blocking"
            fi
```

- [ ] **Step 2: Commit and push**


```go
func (s *ProjectService) HandleDeployment(ctx context.Context, req model.WebhookDeploymentRequest) (*model.ProjectResponse, error) {
	project, err := s.queries.GetProjectByContainerImage(ctx, req.Image)
	if err != nil {
		return nil, fmt.Errorf("project not found for image %s: %w", req.Image, err)
	}

	version := req.NewDigest
	if project.Version.Valid {
		version = project.Version.String
	}

	updated, err := s.queries.UpdateProjectDeployment(ctx, repository.UpdateProjectDeploymentParams{
		Slug:            project.Slug,
		DeployedVersion: pgtype.Text{String: version, Valid: true},
	})
	if err != nil {
		return nil, fmt.Errorf("failed to update deployment: %w", err)
	}

	resp := s.toProjectResponse(updated)
	return &resp, nil
}
```

- [ ] **Step 5: Add Deployment handler**

Add to `internal/handler/webhook.go`:

```go
func (h *WebhookHandler) Deployment(w http.ResponseWriter, r *http.Request) {
	var req model.WebhookDeploymentRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		http.Error(w, `{"error":"invalid request body"}`, http.StatusBadRequest)
		return
	}

	if req.Image == "" {
		http.Error(w, `{"error":"image is required"}`, http.StatusBadRequest)
		return
	}

	resp, err := h.svc.HandleDeployment(r.Context(), req)
	if err != nil {
		http.Error(w, fmt.Sprintf(`{"error":"%s"}`, err.Error()), http.StatusNotFound)
		return
	}

	w.Header().Set("Content-Type", "application/json")
	json.NewEncoder(w).Encode(resp)
}
```
